\begin{abstract}
The fundamental cryptographic assumption defining password security relies on the theoretical search space equation, assuming a uniform distribution of character selection. This paper empirically demonstrates that human-generated passwords exhibit severe structural and linguistic predictability, drastically collapsing the effective search space. By processing a corpus of 1.33 million sanitized credentials through a Probabilistic Context-Free Grammar (PCFG) and a character-level Long Short-Term Memory (LSTM) neural network, we developed a predictive ranking pipeline based on cross-entropy loss. Testing this pipeline against a sequestered validation set of 100,000 unseen passwords revealed critical vulnerabilities. When mapped against the hardware constraints of the memory-hard Argon2id hashing algorithm running on an NVIDIA RTX 4090 (5,000 H/s), the probabilistically sorted pipeline achieved a 30\% total database compromise in exactly 6.00 seconds. These findings indicate that hardware-based throttling is fundamentally insufficient against probabilistically optimized threat actors.
\end{abstract}

\section{Introduction}
Modern password security relies heavily on cryptographic key derivation functions (KDFs) such as bcrypt \cite{provos1999}, scrypt \cite{percival2009}, and Argon2id \cite{argon2}. These algorithms are designed to be memory-hard, purposefully bottlenecking the parallel processing capabilities of modern Graphics Processing Units (GPUs) and Application-Specific Integrated Circuits (ASICs). The defensive strategy is rooted in the mathematical search space equation:

\begin{equation}
S = R^L
\end{equation}

where $S$ is the total search space, $R$ is the character set size, and $L$ is the string length.

If an attacker is forced to search a theoretical space of $S \approx 200$ trillion combinations at a heavily throttled hardware rate, the target is considered mathematically secure. However, this model fails to account for human psychology. Humans operate under strict cognitive limits, relying on semantic memory, spatial keyboard walks, and recycled structural templates \cite{bonneau2012}. This paper proposes that when adversarial machine learning is used to mathematically map these behavioral patterns, the effective search space collapses, rendering hardware throttling obsolete. 

The primary contribution of this paper is the empirical demonstration that advanced deep learning architectures, specifically Long Short-Term Memory (LSTM) networks, can ingest large breach corpora to model the exact conditional probabilities of human keystrokes. By shifting the computational burden to a pre-computation (training) phase, the execution phase of the attack is reduced to an optimized pipeline that bypasses real-time memory-hard defenses.

\section{Threat Model and Theoretical Background}

\subsection{Information Entropy and Zipf's Law}
Entropy, in a Shannon information-theoretic sense \cite{shannon1948}, measures the unpredictability of a state. Cryptographers design systems assuming passwords contain high entropy (e.g., bits generated by a true random number generator). However, human beings are biologically incapable of generating true randomness. 

When prompted to create a password, a user inevitably draws from episodic memory (names, dates) or semantic memory (words, cultural references). Because large populations draw from the exact same cultural lexicons, the statistical distribution of password choices is heavily skewed, closely following Zipf's Law \cite{zipf1949}. Zipf's law states that given a large corpus, the frequency of any word is inversely proportional to its rank in the frequency table. Mathematically:
\begin{equation}
P(r) \sim \frac{1}{r^\alpha}
\end{equation}
where $r$ is the rank of the password and $\alpha$ is a constant characterizing the distribution. This means the vast majority of human choices are concentrated in a microscopic fraction of the theoretical search space $S$.

\subsection{Memory-Hard Defense Mechanisms}
To counter offline brute-force attacks, defenders rely on KDFs like Argon2id. Argon2id utilizes three specific cost parameters to defend the hash:
\begin{itemize}
    \item \textbf{Time Cost ($t$):} The number of iterations.
    \item \textbf{Memory Cost ($m$):} The amount of RAM required to compute the hash (e.g., 64 MB).
    \item \textbf{Parallelism ($p$):} The number of threads used.
\end{itemize}
By forcing a GPU to load substantial data into its limited VRAM for every single guess, the throughput is deliberately bottlenecked. A modern RTX 4090, which can calculate billions of legacy MD5 hashes per second, is reduced to approximately 5,000 Argon2id hashes per second.

\section{Related Work}

The landscape of offline password cracking has evolved significantly from early deterministic methodologies to modern probabilistic models.

\subsection{Deterministic and Rule-Based Cracking}
Early cryptanalysis relied heavily on standard dictionary attacks and rule-based mutations. Tools like Hashcat \cite{hashcat} utilize massive text files combined with rule engines to append digits, toggle cases, and substitute characters. While effective against legacy hashes, these exhaustive methods generate massive amounts of computational waste, rendering them inefficient against memory-hard targets.

\subsection{Markov Models and PCFGs}
To optimize guessing efficiency, researchers began modeling the statistical transition probabilities between characters using Markov chains \cite{narayanan2015}. Weir et al. expanded on this by pioneering the use of Probabilistic Context-Free Grammars (PCFGs) to map the structural syntax of passwords \cite{weir2009}. PCFGs successfully mapped structural templates (e.g., the high probability of a password ending in two digits), but struggled to maintain deep semantic context across longer strings.

\subsection{Deep Learning and Generative Adversarial Networks}
Recently, the introduction of deep learning has revolutionized offensive security. Hitaj et al. introduced PassGAN \cite{hitaj2019}, utilizing Generative Adversarial Networks to autonomously generate password guesses. Similarly, Melicher et al. utilized neural language models to predict guessability \cite{melicher2016}. 

Our research expands on these neural foundations. Instead of utilizing GANs to passively generate lists, we engineered a specific character-level LSTM pipeline designed to calculate exact cross-entropy perplexity scores. This allows us to actively rank an unseen validation database, creating an aggressively optimized attack pipeline specifically designed to circumvent Argon2id hardware constraints.

\subsection{Neural Architecture Deep-Dive}

To move beyond basic structural skeletons, we engineered a character-level Long Short-Term Memory (LSTM) network utilizing the PyTorch framework \cite{paszke2019}. 

\subsubsection{The Embedding Layer}
Raw characters cannot be fed directly into a neural network. Therefore, an embedding layer was constructed to map the discrete 96-character ASCII vocabulary into a continuous dense vector space. Let $v$ be the vocabulary size and $d$ be the embedding dimension (set to 128). The embedding matrix $E \in \mathbb{R}^{v \times d}$ transforms a character index $c_i$ into a dense vector $e_i = E(c_i)$. This allows the network to learn spatial and semantic relationships between characters (e.g., learning that '1' and '2' are frequently sequential).

\subsubsection{LSTM Memory Gates}
Unlike standard feed-forward networks, LSTMs contain internal memory states that allow them to understand sequential context. The LSTM updates its hidden state $h_t$ based on the current embedded character $e_t$ and the previous state $h_{t-1}$. Internally, this is governed by three specific gates:
\begin{align}
f_t &= \sigma(W_f \cdot [h_{t-1}, e_t] + b_f) \quad \text{(Forget Gate)} \\
i_t &= \sigma(W_i \cdot [h_{t-1}, e_t] + b_i) \quad \text{(Input Gate)} \\
o_t &= \sigma(W_o \cdot [h_{t-1}, e_t] + b_o) \quad \text{(Output Gate)}
\end{align}
\begin{equation}
c_t = f_t * c_{t-1} + i_t * \tanh(W_c \cdot [h_{t-1}, e_t] + b_c)
\end{equation}

This mathematical structure allows the AI to "learn" the grammar of human heuristics, calculating the conditional probability of the next character:
\begin{equation}
P(x_t | x_{t-1}, x_{t-2}, \dots, x_1)
\end{equation}

\subsection{The Scoring Engine}
Following training, the model evaluated the 100,000 unseen validation passwords. For each string in the validation set, the network calculated the cross-entropy loss. A low loss score indicates high mathematical predictability, while a high loss score indicates severe unpredictability. 

\begin{algorithm}
\caption{Neural Ranked Guessing Pipeline}
\begin{algorithmic}[1]
\REQUIRE $V$: Validation Set (100,000 passwords)
\REQUIRE $M$: Trained LSTM Model
\ENSURE $S$: Sorted Attack Pipeline
\STATE Initialize empty list $S$
\FOR{each $p \in V$}
    \STATE $L_{total} \leftarrow 0$
    \FOR{each character $c_i \in p$}
        \STATE $P_{pred} \leftarrow M(c_1 \dots c_{i-1})$
        \STATE $L_{total} \leftarrow L_{total} - \log(P_{pred}(c_i))$
    \ENDFOR
    \STATE $Score \leftarrow L_{total} / length(p)$
    \STATE Append $(p, Score)$ to $S$
\ENDFOR
\STATE Sort $S$ in ascending order based on $Score$
\RETURN $S$
\end{algorithmic}
\end{algorithm}

\section{Defensive Countermeasures}
Given that offline AI sorting neutralizes offline hardware throttling, defensive paradigms must shift. We propose alignment with NIST Special Publication 800-63B guidelines \cite{nist80063b}. Systems should abandon arbitrary complexity rules (which encourage predictable templates like \texttt{ULLLLLLD}) and instead implement neural-based pre-screening at the time of password creation. Integrating algorithms like Dropbox's Zxcvbn \cite{wheeler2016} allows servers to reject low-perplexity choices before they are hashed. Furthermore, transitioning to zero-knowledge proofs and FIDO2/WebAuthn physical keys eliminates the human cognitive factor entirely.

\section{Ethical Considerations}
This research was conducted exclusively for offensive security validation and defensive advancement. All neural networks were trained on publicly available, sanitized data breaches (RockYou). No active network infrastructure or proprietary databases were targeted or compromised during this study.

\section{Limitations}
While the PCFG-LSTM pipeline is highly lethal against human psychology, it provides zero statistical advantage against mathematical randomness. Passwords generated by cryptographic password managers successfully survived the targeted attack bounds. Furthermore, the model's predictive capabilities are inherently bound to the cultural and linguistic biases present in the specific training corpus (e.g., English-centric datasets).

\section{Conclusion}
The security margin of a password cannot be accurately measured by its length or the mathematical boundaries of its character space. Human beings do not select characters uniformly at random. By processing breach corpora through character-level neural networks, attackers can empirically quantify linguistic and structural predictability, drastically collapsing the effective search space.

Our results demonstrate that this neural sorting allows an attacker to completely bypass the hardware limitations of memory-hard KDFs like Argon2id. Compromising 30\% of a target dataset in under six seconds illustrates that current cryptographic cost parameters are drastically undersized when faced with AI-optimized threat actors. To properly defend against these pipelines, security paradigms must shift away from theoretical entropy and begin dynamically evaluating credentials based on neural perplexity at the time of creation.

class PasswordLSTM(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, num_layers):
        super(PasswordLSTM, self).__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)

def evaluate_passwords(val_passwords, model, char2int, vocab_size):
    criterion = nn.CrossEntropyLoss(ignore_index=char2int['<PAD>'], reduction='none')
    password_scores = []
    
    with torch.no_grad():
        for p in val_passwords:
            encoded = [char2int.get(c, char2int['<PAD>']) for c in p]
            inputs = torch.tensor([encoded[:-1]], dtype=torch.long).to(device)
            targets = torch.tensor([encoded[1:]], dtype=torch.long).to(device)

import re
from collections import Counter

def sanitize_and_tokenize(password_list):
    """
    Sanitizes raw passwords and extracts PCFG skeletons.
    L = Lowercase, U = Uppercase, D = Digit, S = Symbol
    """
    skeleton_counts = Counter()
    clean_passwords = []
    
    for pwd in password_list:
        # Step 1: Filter length and ASCII compliance
        if 8 <= len(pwd) <= 24 and pwd.isascii():
            clean_passwords.append(pwd)
            
            # Step 2: PCFG Tokenization
            skeleton = ""
            for char in pwd:
                if char.islower():
                    skeleton += 'L'
                elif char.isupper():
                    skeleton += 'U'
                elif char.isdigit():
                    skeleton += 'D'
                else:
                    skeleton += 'S'
            
            skeleton_counts[skeleton] += 1
            
    return clean_passwords, skeleton_counts.most_common(5)
